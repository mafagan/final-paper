\chapter{总结与展望}
\section{本文总结}
近年来，本体领域的研究者对存在规则的容忍不一致查询问题的关注度越来越高。Lukasiewicz~等人研究了在几类扩充了否定限制的存在规则片段下~AR~语义回答查询的数据复杂度\cite{ecai2012Lukasiewicz,otm2013Lukasiewicz}，并在\cite{bworld}中深入分析了相关问题的组合复杂度。与这个问题相关的另外一个重要工作是从描述逻辑语言族发展过来的~AR~与~IAR~语义。最近的工作包括\cite{ijcai2011Rosati,aaai2012Bienvenu,ijcai2013Bienvenu}都是在研究~AR~与~IAR~语义下的回答查询。另外与此相关的还有基于带偏好的~AR~与~IAR。

除了修改数据集得到修复外，同样有与本文类似的修改规则的研究，Scharrenbach~说明了概率描述逻辑可以用于解决冲突并获得一致的知识基，从而使得推理变得有效\cite{dlog2010Scharrenbach}。Qi~与~Du~等人为描述逻辑下的~terminologies~提出了基于模型的修正算子\cite{ijcai2009QiD}，而~Wang~则为本体修正提出了“理论模型”方法\cite{dlog2014Wang}。为了解决因为不一致而产生的本体推理不确定性，Gottlob等用马尔科夫逻辑网络对~Datalog$\pm$~进行了不确定性概率的扩展\cite{amai2013Gottlob}。另外还有几项工作是基于不一致本体推理的研究，如\cite{ijcai2005Huang,semweb2005Haase}。

我们建立了一个用于处理稳定模型语义下不一致存在规则本体的理论框架，在这个框架下，我们分析了容忍不一致查询问题的数据复杂度与组合复杂度。同时我们提出了通过调用~Q(SAT)~或~ASP~来解决回答查询的实际求解方案，并且实现了一个叫做~QAIER~的原型系统。在这个系统下的实验表明，基于规则修复的查询回答问题在实际应用中有着良好的可扩展性。
\section{后续工作及展望}
本文的后续研究可以从三个方向进行描述。
\begin{enumerate}
\item 技术优化

本文提出的稳定模型语义下规则修复框架是一个新理论框架，本文对该框架下的问题研究还不够彻底深入，全面，基于规则的偏好修复还有非常多的性质可以挖掘。
通过我们的实验数据中可以发现，虽然理论上有数据复杂度为多项式的，但规模与效率距离实际应用仍然有非常大的距离。
所以发现可优化的片段并设计对应的有效算法，寻找规则修复的特殊性质并改进算法是后续具有挑战性及有意义的工作。
另外，根据实际情况模拟新的查询语义也有可能是有效且快速的改进我们问题复杂度的方向，当前定义的查询语义即所有的修复都必须蕴含查询也许太过简单直接。
\item 可重写片段

Calvanese~等人提出了基于本体的数据访问(OBDA: Ontolgy Based Data Access)将本体与传统数据库技术相结合，使传统数据库具备逻辑推理能力\cite{Cal2007}。基于本体数据访问可以应用与数据查询、数据集成和数据交换等领域，并得到各大数据库厂商和开发组织的表达能力。Calvanese~等人证明了~DL-Lite~下的合取查询具有一阶可重写性(First Order Rewritable)，即给定一个本体~$\Sigma$，查询~Q，Q~和~$\Sigma$~可以被重写成一阶查询语句~$r_{Q,\Sigma}$\cite{CalARJ2007}。Abiteboul~等人(1995)首先提出传统的一阶查询可以被重写成SQL语句，即对于(布尔)查询~Q，可以根据本体~$\Sigma$~及其到数据库表的映射~M~，将~Q~重写成~SQL~语句，并在传统数据库上进行查询\cite{Serge1995}。经过一阶重写转换，原来的合取查询可以转化成计算复杂度为~\textsc{AC0}~的~SQL~查询，有效降低了查询复杂度。不同片段下的重写算法也是目前研究的热点问题。Calì等(2010)提出了一种针对非~Guarded~的存在规则语言片段的~Rewriting~方法\cite{CalAMW2010}。Gottlob等(2011)将这一方法扩展到~Linear~存在规则语言片段\cite{GottlobICDE2011}。
所以在基于规则修复的框架下研究一阶可重写片段，并提出重写方法能有效的提高我们求解问题的效率。
\item 回答查询求解器

查询回答求解器对于推动基于本体的数据访问广泛应用的关键，长久以来限制基于本体数据访问被广泛使用的一个主要瓶颈就在于缺乏有效的求解器，目前存在语言查询回答工具仅如表\ref{fig:tools}所示。

	\includegraphics[width=400pt]{img/tools.png}\\
	\caption{主要的存在规则语言工具} \label{fig:tools}

因此，我们很有必要在提出稳定模型语义下不一致本体存在规则语言修复理论后，实现高效的求解器。
本文实验中所实现的原型系统在仍十分不完善，代码未经过高度优化，所使用的技术又较为单一，所以将其进行改进并向着求解器的方向发展可作为下一步的重要工作。
\end{enumerate}
